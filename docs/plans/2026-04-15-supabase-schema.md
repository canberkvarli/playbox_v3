# Supabase Schema + RLS Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Move Playbox v3 from in-memory seed data to a Supabase backend with PostGIS-backed stations, per-user session tracking, follows, and profile stats — all gated by Clerk-JWT-verified RLS.

**Architecture:** Supabase Postgres + PostGIS for geo queries. Auth is delegated to Clerk via a `supabase` JWT template signed with Supabase's JWT secret (HS256), so `auth.jwt() ->> 'sub'` inside policies matches Clerk's user ID. App reads via TanStack Query hooks wrapping the existing `useSupabase()` client. Schema is versioned with Supabase CLI migrations checked into git. The 30-row seed ships as a seed SQL file; the demo-generator fallback (`lib/generateStations.ts`) stays for users outside Turkey.

**Tech Stack:** Supabase (Postgres 15 + PostGIS), Clerk (JWT template), Supabase CLI, TanStack Query v5, Expo SDK 52, TypeScript.

---

## Why RLS matters (read this before skipping it)

The Supabase anon key ships **inside the Expo bundle** — anyone can extract it and hit the REST API directly. RLS is what makes the Clerk JWT actually enforce ownership. Without it:
- Any user can read every other user's session history (when + where they play).
- Any user can insert fake `sessions` rows under someone else's `user_id`.
- Any user can forge `follows` edges.

Policies that check `auth.jwt() ->> 'sub' = user_id` are the entire security model. For `stations` (public read) RLS is minor; for `sessions`/`follows`/`user_profiles` it's load-bearing.

  city = excluded.city,
  location = excluded.location,
  sports = excluded.sports,

## Task 8: `user_stats` view (total minutes, streak, city rank)

**Files:**
- Create: `supabase/migrations/20260415000006_user_stats.sql`

**Step 1: Write the migration**

```sql
-- 20260415000006_user_stats.sql

-- Total minutes per user, based on ended sessions only.
create view user_stats as
with minutes as (
  select
    user_id,
    coalesce(
      sum(extract(epoch from (ended_at - started_at)) / 60.0),
      0
    )::int as total_minutes
  from sessions
  where ended_at is not null
  group by user_id
),
streaks as (
  -- A day "counts" if user has >= 1 ended session that day (Europe/Istanbul).
  -- Streak = number of consecutive days up to today where this holds.
  select
    user_id,
    count(*) as streak_days
  from (
    select
      user_id,
      date_trunc('day', started_at at time zone 'Europe/Istanbul')::date as d,
      row_number() over (
        partition by user_id
        order by date_trunc('day', started_at at time zone 'Europe/Istanbul') desc
      ) as rn
    from sessions
    where ended_at is not null
    group by user_id, d
  ) s
  where (current_date - d) = (rn - 1)  -- contiguous from today
  group by user_id
),
city_rank as (
  select
    p.user_id,
    p.city,
    rank() over (
      partition by p.city
      order by coalesce(m.total_minutes, 0) desc
    ) as city_rank
  from user_profiles p
  left join minutes m on m.user_id = p.user_id
)
select
  p.user_id,
  coalesce(m.total_minutes, 0) as total_minutes,
  coalesce(s.streak_days, 0)   as streak_days,
  cr.city_rank
from user_profiles p
left join minutes    m  on m.user_id  = p.user_id
left join streaks    s  on s.user_id  = p.user_id
left join city_rank  cr on cr.user_id = p.user_id;

-- Views inherit RLS from base tables, but sessions is self-only.
-- To let users see their OWN stats, we expose via a security-definer function.
create or replace function my_stats()
returns table (total_minutes int, streak_days int, city_rank bigint)
language sql
security definer
set search_path = public
as $$
  select total_minutes, streak_days, city_rank
  from user_stats
  where user_id = auth.jwt() ->> 'sub';
$$;

revoke all on function my_stats() from public;
grant execute on function my_stats() to authenticated;
```

## Task 9: Nearby-stations RPC

Keeps PostGIS query logic server-side so the client doesn't need `ST_DWithin` expertise.

**Files:**
- Create: `supabase/migrations/20260415000007_stations_nearby_rpc.sql`

**Step 1: Write the migration**

```sql
-- 20260415000007_stations_nearby_rpc.sql
create or replace function stations_nearby(
  lat double precision,
  lng double precision,
  radius_km double precision default 10
)
returns table (
  id            text,
  name          text,
  city          city,
  lat           double precision,
  lng           double precision,
  sports        sport[],
  stock         jsonb,
  available_now boolean,
  distance_m    double precision
)
language sql
stable
as $$
  select
    s.id,
    s.name,
    s.city,
    ST_Y(s.location::geometry) as lat,
    ST_X(s.location::geometry) as lng,
    s.sports,
    s.stock,
    s.available_now,
    ST_Distance(
      s.location,
      ST_SetSRID(ST_MakePoint(lng, lat), 4326)::geography
    ) as distance_m
  from stations s
  where ST_DWithin(
    s.location,
    ST_SetSRID(ST_MakePoint(lng, lat), 4326)::geography,
    radius_km * 1000
  )
  order by distance_m asc;
$$;

grant execute on function stations_nearby(double precision, double precision, double precision) to anon, authenticated;
```

## Task 12: `useStations` hook

**Files:**
- Create: `hooks/useStations.ts`
- Test: (no unit test — covered by manual smoke; TanStack hooks need RN Testing Library setup which is out of scope here)

**Step 1: Write the hook**

```ts
// hooks/useStations.ts
import { useQuery } from '@tanstack/react-query';
import { useSupabase } from '@/lib/supabase';
import type { Station, Sport } from '@/data/stations.seed';

type NearbyRow = {
  id: string;
  name: string;
  city: Station['city'];
  lat: number;
  lng: number;
  sports: Sport[];
  stock: Partial<Record<Sport, number>>;
  available_now: boolean;
  distance_m: number;
};

export function useStations(near: { lat: number; lng: number } | null, radiusKm = 10) {
  const supabase = useSupabase();

  return useQuery<Station[]>({
    queryKey: ['stations', near?.lat, near?.lng, radiusKm],
    enabled: !!supabase && !!near,
    queryFn: async () => {
      if (!supabase || !near) return [];
      const { data, error } = await supabase.rpc('stations_nearby', {
        lat: near.lat,
        lng: near.lng,
        radius_km: radiusKm,
      });
      if (error) throw error;
      return (data as NearbyRow[]).map((r) => ({
        id: r.id,
        name: r.name,
        city: r.city,
        lat: r.lat,
        lng: r.lng,
        sports: r.sports,
        stock: r.stock,
        availableNow: r.available_now,
      }));
    },
  });
}
```

import { useEffect } from 'react';
import { useUser } from '@clerk/clerk-expo';
import { useSupabase } from '@/lib/supabase';

export function useEnsureProfile() {
  const { user, isLoaded } = useUser();
  const supabase = useSupabase();

  useEffect(() => {
    if (!isLoaded || !user || !supabase) return;
    const displayName = [user.firstName, user.lastName].filter(Boolean).join(' ') || 'Oyuncu';
    const username = user.username || `p_${user.id.slice(-8)}`;

    supabase
      .from('user_profiles')
      .upsert(
        { user_id: user.id, display_name: displayName, username },
        { onConflict: 'user_id', ignoreDuplicates: true }
      )
      .then(({ error }) => {
        if (error && __DEV__) console.warn('[ensure profile]', error);
      });
  }, [isLoaded, user, supabase]);
}

// hooks/useStartSessionMutation.ts
import { useMutation } from '@tanstack/react-query';
import { useSupabase } from '@/lib/supabase';
import type { Sport } from '@/data/stations.seed';

type Vars = { stationId: string; sport: Sport; plannedMinutes: number };

export function useStartSessionMutation() {
  const supabase = useSupabase();
  return useMutation({
    mutationFn: async ({ stationId, sport, plannedMinutes }: Vars) => {
      if (!supabase) throw new Error('supabase not ready');
      const { data, error } = await supabase
        .from('sessions')
        .insert({
          station_id: stationId,
          sport,
          planned_minutes: plannedMinutes,
        })
        .select('id, started_at')
        .single();
      if (error) throw error;
      return data as { id: number; started_at: string };
    },
  });
}
```

import { useMutation } from '@tanstack/react-query';
import { useSupabase } from '@/lib/supabase';

export function useEndSessionMutation() {
  const supabase = useSupabase();
  return useMutation({
    mutationFn: async () => {
      if (!supabase) throw new Error('supabase not ready');
      // Close the user's single active session (partial unique index guarantees one).
      const { error } = await supabase
        .from('sessions')
        .update({ ended_at: new Date().toISOString() })
        .is('ended_at', null);
      if (error) throw error;
    },
  });
}
```

import { useQuery } from '@tanstack/react-query';

export function useMyStats() {
  const supabase = useSupabase();

    queryKey: ['my_stats'],

## Risk notes

- **JWT template misconfig** is the #1 failure mode. If `sub` doesn't match between Clerk and what policies check, everything silently returns empty. Task 10 step 4 is the smoke test — don't skip it.
- **Partial unique index on `sessions`** will reject concurrent unlocks with a 23505 error code. Client code in Task 14 doesn't handle this yet — v2 should surface a toast ("Zaten aktif bir oturumunuz var").
- **`my_stats()` uses `security definer`** — review the function body once more before shipping; it only selects from `user_stats` filtered by `auth.jwt() ->> 'sub'`, so no privilege escalation, but any future change to that function needs a second set of eyes.
- **PostGIS `ST_DWithin` on `geography`** is accurate but slow without the GIST index in Task 3. Verify the index is present before assuming perf is fine at 10k+ rows.
